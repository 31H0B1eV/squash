1 User Guide
2 ==========
3 
4 While the Squash API is fairly large, in general you will only need a
5 handful of functions.
6 
7 Squash's API follows some pretty standard conventions:
8 
9 - Type names are CamelCase (with the first letter uppercase)
10 - Functions are lowercase_with_underscores
11 - Binary data is two variablesâ€”a `uint8_t*` and a `size_t`.
12 - Instance arguments come first, followed by any output arguments,
13  followed by input arguments.
14 
15 The heart of Squash is the [codec](@ref SquashCodec). A codec is an
16 implementation of an algorithm, and codecs with the same name are
17 assumed to be compatible. A single [plugin](@ref SquashPlugin) can,
18 and often does, provide an implementation of several different codecs.
19 
20 Typically you will not deal with [contexts](@ref SquashContext) or
21 plugins directly, and you may only deal with strings to represent
22 codecs.

24 @section buffers Buffer API
25 
26 If you have a block of data in memory which you want to compress (or
27 decompress), the easiest way to do so is using the buffer API.
28 
29 ~~~{.c}
30 SquashStatus
31 squash_compress (const char* codec,
32  uint8_t* compressed, size_t* compressed_length,
33  const uint8_t* uncompressed, size_t uncompressed_length, ...);
34 ~~~
35 
36 Like many functions in Squash, ::squash_compress returns a
37 [SquashStatus](@ref SquashStatus). If the operation was successful, a
38 positive number is returned (generally ::SQUASH_OK), and a negative
39 number is returned on failure.
40 
41 The first argument is the name of the codec.
42 
43 The @a compressed and @a compressed_length arguments represent the
44 buffer which you wish to compress the data into. Notice that @a
45 compressed_length is a `size_t*`, not a `size_t`. When you call the
46 function it should be a pointer to the number of bytes available for
47 writing in the compressed buffer, and the function will alter the
48 value to the true size of the compressed data upon successful
49 compression.
50 
51 @a uncompressed and @a uncompressed_length are the buffer you wish to
52 compress. @a uncompressed_length is a `size_t`, not a `size_t`,
53 because the function does not need to modify the value.
54 
55 Finally, this function is variadicâ€”it accepts an arbitrary number of
56 options, which you can use to control things like the compression
57 level. Options are key/value pairs of strings, terminated by a *NULL*
58 sentinelâ€¦ More on that later, but for now, just know that passing
59 *NULL* there will use the defaults, which is generally what you want.
60 
61 Knowing how big the @a compressed buffer needs to be generally
62 requires another function:
63 
64 ~~~{.c}
65 size_t
66 squash_get_max_compressed_size (const char* codec, size_t uncompressed_length);
67 ~~~
68 
69 This function returns the maximum buffer size necessary to hold @a
70 uncompressed_length worth of compressed data, even in the worst case
71 scenario (uncompressable data). Typically it is slightly larger than
72 @a uncompressed_length.
73 
74 So, if you wanted to compress some data using the deflate algorithm,
75 you end up with something like this:
76 
77 ~~~{.c}
78 const char* uncompressed = "Hello, world!";
79 size_t uncompressed_length = strlen (uncompressed);
80 size_t compressed_length = squash_get_max_compressed_size ("deflate", uncompressed_length);
81 uint8_t* compressed = (uint8_t*) malloc (compressed_length);
82 
83 SquashStatus res =
84  squash_compress ("deflate",
85  compressed, &compressed_length,
86  (const uint8_t*) uncompressed, uncompressed_length,
87  NULL);
88 
89 if (res != SQUASH_OK) {
90  fprintf (stderr, "Unable to compress data [%d]: %s\n",
91  res, squash_status_to_string (res));
92  exit (res);
93 }
94 
95 fprintf (stdout, "Compressed a %lu byte buffer to %lu bytes.\n",
96  uncompressed_length, compressed_length);
97 ~~~
98 
99 Decompression is basically the same:
100 
101 ~~~{.c}
102 size_t decompressed_length = uncompressed_length + 1;
103 char* decompressed = (char*) malloc (uncompressed_length + 1);
104 
105 res = squash_decompress ("deflate",
106  (uint8_t*) decompressed, &decompressed_length,
107  compressed, compressed_length, NULL);
108 
109 if (res != SQUASH_OK) {
110  fprintf (stderr, "Unable to decompress data [%d]: %s\n",
111  res, squash_status_to_string (res));
112  exit (res);
113 }
114 
115 /* Notice that we didn't compress the *NULL* byte at the end of the
116  string. We could have, it's just a waste to do so. */
117 decompressed[decompressed_length] = '\0';
118 
119 if (strcmp (decompressed, uncompressed) != 0) {
120  fprintf (stderr, "Bad decompressed data.\n");
121  exit (-1);
122 }
123 
124 fprintf (stdout, "Successfully decompressed.\n");
125 ~~~
126 
127 It is worth noting that "Hello, world!" is really too short to
128 compress (it "compresses" from 13 bytes to 15 bytes here), but using a
129 longer string will yield better results. You can find a complete,
130 self-contained example in [simple.c](@ref simple.c).

132 @section streams Stream API
133 
134 While the buffer API is very easy to use, if you want to compress a
135 lot of data without loading it into memory then the stream API is much
136 more appropriate.
137 
138 If you've ever worked with a compression API in the past then the
139 stream API will probably feel pretty familiar to you. The basic idea
140 is that you have a data structure, a @ref SquashStream, and you
141 shuttle data into one buffer and out of another.
