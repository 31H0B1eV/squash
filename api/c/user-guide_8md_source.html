1 User Guide
2 ==========
3 
4 While the Squash API is fairly large, it is conceptually fairly
5 simple.
6 
7 Most of the functions are really just layers of convenience wrappers
8 which follow common conventions; for example, many functions will have
9 versions for string and @ref SquashCodec parameters for the codec, and
10 variadic arguments or a @ref SquashOptions parameter for the options,
11 which means four versions for every function, but only one concept to
12 learn.
13 
14 Squash's API follows some pretty standard conventions:
15 
16 - Type names are CamelCase (with the first letter uppercase)
17 - Functions are lowercase_with_underscores
18 - Binary data is two parametersâ€”a `size_t` followed by a `uint8_t*`.
19 - Instance arguments come first, followed by any output arguments,
20  followed by input arguments.
21 
22 The heart of Squash is the [codec](@ref SquashCodec). A codec is an
23 implementation of an algorithm, and codecs with the same name are
24 assumed to be compatible. A single [plugin](@ref SquashPlugin) can,
25 and often does, provide an implementation of several different codecs.
26 
27 Typically you will not deal with [contexts](@ref SquashContext) or
28 plugins directly, and you may only deal with strings to represent
29 codecs.

31 @section buffers Buffer API
32 
33 If you have a block of data in memory which you want to compress (or
34 decompress), the easiest way to do so is using the buffer API.
35 
36 ~~~{.c}
37 SquashStatus
38 squash_compress (const char* codec,
39  size_t* compressed_length,
40  uint8_t compressed[],
41  size_t uncompressed_length,
42  const uint8_t uncompressed[],
43  ...);
44 ~~~
45 
46 Like many functions in Squash, ::squash_compress returns a
47 [SquashStatus](@ref SquashStatus). If the operation was successful, a
48 positive number is returned (generally ::SQUASH_OK), and a negative
49 number is returned on failure.
50 
51 The first argument is the name of the codec. To list available
52 codecs, you can use the `squash -L` command; the convention is that
53 codecs are just the lowercase name of the algorithm. For example,
54 "gzip", "lz4", and "bzip2" are all what you would probably expect.
55 
56 The @a compressed and @a compressed_length arguments represent the
57 buffer which you wish to compress the data into. Notice that @a
58 compressed_length is a `size_t*`, not a `size_t`. When you call the
59 function it should be a pointer to the number of bytes available for
60 writing in the compressed buffer, and the function will alter the
61 value to the true size of the compressed data upon successful
62 compression.
63 
64 @a uncompressed and @a uncompressed_length are the buffer you wish to
65 compress. @a uncompressed_length is a `size_t`, not a `size_t`,
66 because the function does not need to modify the value.
67 
68 Finally, this function is variadicâ€”it accepts an arbitrary number of
69 options, which you can use to control things like the compression
70 level. Options are key/value pairs of strings, terminated by a *NULL*
71 sentinelâ€¦ More on that later, but for now, just know that passing
72 *NULL* there will use the defaults, which is generally what you want.
73 
74 Knowing how big the @a compressed buffer needs to be generally
75 requires another function:
76 
77 ~~~{.c}
78 size_t
79 squash_get_max_compressed_size (const char* codec, size_t uncompressed_length);
80 ~~~
81 
82 This function returns the maximum buffer size necessary to hold @a
83 uncompressed_length worth of compressed data, even in the worst case
84 scenario (uncompressable data). Typically it is slightly larger than
85 @a uncompressed_length.
86 
87 So, if you wanted to compress some data using the deflate algorithm,
88 you end up with something like this:
89 
90 ~~~{.c}
91 const char* uncompressed = "Hello, world!";
92 size_t uncompressed_length = strlen (uncompressed);
93 size_t compressed_length = squash_get_max_compressed_size ("deflate", uncompressed_length);
94 uint8_t* compressed = (uint8_t*) malloc (compressed_length);
95 
96 SquashStatus res =
97  squash_compress ("deflate",
98  &compressed_length, compressed,
99  uncompressed_length, (const uint8_t*) uncompressed,
100  NULL);
101 
102 if (res != SQUASH_OK) {
103  fprintf (stderr, "Unable to compress data [%d]: %s\n",
104  res, squash_status_to_string (res));
105  exit (EXIT_FAILURE);
106 }
107 
108 fprintf (stdout, "Compressed a %lu byte buffer to %lu bytes.\n",
109  uncompressed_length, compressed_length);
110 ~~~
111 
112 Decompression is basically the same:
113 
114 ~~~{.c}
115 size_t decompressed_length = uncompressed_length + 1;
116 char* decompressed = (char*) malloc (uncompressed_length + 1);
117 
118 res = squash_decompress ("deflate",
119  &decompressed_length, (uint8_t*) decompressed
120  compressed_length, compressed, NULL);
121 
122 if (res != SQUASH_OK) {
123  fprintf (stderr, "Unable to decompress data [%d]: %s\n",
124  res, squash_status_to_string (res));
125  exit (EXIT_FAILURE);
126 }
127 
128 /* Notice that we didn't compress the *NULL* byte at the end of the
129  string. We could have, it's just a waste to do so. */
130 decompressed[decompressed_length] = '\0';
131 
132 if (strcmp (decompressed, uncompressed) != 0) {
133  fprintf (stderr, "Bad decompressed data.\n");
134  exit (EXIT_FAILURE);
135 }
136 
137 fprintf (stdout, "Successfully decompressed.\n");
138 ~~~
139 
140 It is worth noting that "Hello, world!" is really too short to
141 compress (it "compresses" from 13 bytes to 15 bytes here), but using a
142 longer string will yield better results. You can find a complete,
143 self-contained example in [simple.c](@ref simple.c).

145 @section streams File I/O API
146 
147 While the buffer API is very easy to use it can be a bit limiting. If
148 you want to compress a lot of data without loading it into memory then
149 a streaming API is much more appropriate.
150 
151 Squash has two streaming APIs; the first is modeled after the API
152 provided by zlib, which has been copied by many other libraries. It
153 is a bit of a pain to use, but extremely powerful. The second API is
154 a higher-level convenience API which wraps the lower-level API and
155 provides an interface similar to the standard C I/O API (*i.e.* fopen,
156 fclose, fflush, fread, and fwrite).
157 
158 We will not go into detail about the lower-level API now, but if
159 you're interested you can look at the API documentation for @ref
160 SquashStream for details.
161 
162 The higher level API is documented in @ref SquashFile, but some of the
163 prototypes are reproduced here:
164 
165 ~~~{.c}
166 SquashFile*
167 squash_file_open (const char* filename,
168  const char* mode,
169  const char* codec,
170  ...);
171 SquashStatus
172 squash_file_read (SquashFile* file,
173  size_t* decompressed_length,
174  uint8_t decompressed[]);
175 SquashStatus
176 squash_file_write (SquashFile* file,
177  size_t uncompressed_length,
178  const uint8_t uncompressed[]);
179 SquashStatus
180 squash_file_flush (SquashFile* file);
181 SquashStatus
182 squash_file_close (SquashFile* file);
183 ~~~
184 
185 Like most of the Squash API, most of these functions return @ref
186 SQUASH_OK on success and a negative error code on failure. The
187 exception to this is @ref squash_file_open which, like `fopen`,
188 returns *NULL* on failure. The @a mode string is passed verbatim to
189 `fopen`.
190 
191 Going back to our "Hello, world!" example from above, to write a file
192 with the compressed version of "Hello, world!", all you would have to
193 do is something like:
194 
195 ~~~{.c}
196 const char* uncompressed = "Hello, world!";
197 
198 SquashFile* file = squash_file_open ("hello.gz", "w+", "gzip", NULL);
199 if (file == NULL) {
200  fprintf (stderr, "Unable to open file for writing\n");
201  exit (EXIT_FAILURE);
202 }
203 
204 SquashStatus res = squash_file_write (file, strlen (uncompressed), uncompressed);
205 if (res != SQUASH_OK) {
206  fprintf (stderr, "Unable to write compressed data [%d]: %s\n",
207  res, squash_status_to_string (res));
208  exit (EXIT_FAILURE);
209 }
210 
211 SquashStatus res = squash_file_close (file);
212 if (res != SQUASH_OK) {
213  fprintf (stderr, "Unable to close file [%d]: %s\n",
214  res, squash_status_to_string (res));
215  exit (EXIT_FAILURE);
216 }
217 ~~~
218 
219 If you want to simply splice the contents of one file to another
220 (decompressing of compressing in the process), you can use the
221 `squash_splice` family of functions, which looks like:
222 
223 ~~~{.c}
224 SquashStatus
225 squash_splice (const char* codec,
226  SquashStreamType stream_type,
227  FILE* fp_out,
228  FILE* fp_in,
229  size_t length,
230  ...);
231 ~~~
232 
233 Beyond just being convenient, this function has the advantage that for
234 codecs which don't implement streaming natively Squash will attempt to
235 memory map the files and pass those addresses directly to the
236 buffer-to-buffer compression function. This eliminates a lot of
237 buffering which, for larger files, can consume significant amounts of
238 memory.
