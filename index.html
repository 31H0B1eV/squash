Squash is primarily an abstraction layer which provides a single API to access many compression libraries. This allows applications a great deal of flexibility in which choosing a compression algorithm, or allowing a choice between several of them.

The actual integration with individual compression libraries is done through plugins which can be installed separately from Squash itself and are not loaded until they are required. This allows Squash consumers to utilize a great many compression algorithms without rewriting code or unnecessary bloat.

In addition to a permissively licensed library, Squash provides some generic tools which use the library for common tasks. Currently, this includes a command-line interface similar to gzip, bzip2, xz, etc. called squash and a benchmarking tool.

Documentation

The C API Reference provides extensive documentation of the C API. For other languages, see the Bindings section.

For the squash command line tool, see the squash(1) man page

Plugins

Squash currently has plugins for the following libraries:

  • bzip2
  • Doboz
  • FastLZ
  • Intel® Performance Primitives
  • LZ4
  • liblzma
  • LZMAT
  • LZF
  • liblzg
  • LZO
  • QuickLZ
  • SHARC
  • Snappy
  • wfLZ
  • zlib
  • ZPAQ

Because Squash provides a single API, any algorithms which Squash has a plugin to support will be useable from any language for which Squash provides bindings.

Squash provides benchmarks to help you decide which codecs are appropriate for your library.

Bindings

Squash is written in C, so it should be possible to create bindings for virtually any programming language. Currently, Squash has bindings for the following languages:

  • C — API Reference
  • Vala

If you're interested in helping to add support for more languages, please get in touch with us.

Benchmarks

Squash ships with a tool for automatically building benchmarks, so you can easily create your own using whatever dataset is appropriate for your work load. To use it with the default datasets, simply type make benchmark.html in the benchmark/ directory of Squash.

If you don't want to run them yourself results from a few systems are posted here. Currently, the these benchmarks are run on two datasets:

iliad (1308638 bytes)

This is the HTML from a copy of Homer's Iliad as downloaded from Project Gutenberg.

enwik8 (100000000 bytes)

The first 100,000,000 bytes of a dump of the English language Wikipedia 2006-03-03. This is one of the datasets from Matt Mahoney's Large Text Compression Benchmark.

Licensing

Squash is licensed under the MIT License, which is highly permissive and allows Squash to be integrated into any application virtually without restriction. That said, please keep in mind that some plugins use libraries which are subject to more restrictive terms (such as the GNU GPL), and using Squash as an intermediary does not release you from the obligations of those licenses if you choose to actually use plugins which use those libraries. Each plugin's page in the C documentation lists the license of the library, or libraries, it uses.

Several of the GPL libraries used by Squash plugins are also available for integration with proprietary software under commercial licenses.
